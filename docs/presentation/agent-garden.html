3 шинэ хүснэгт

shop_agents

Per-shop асаалт + config + метрик.

  • enabled, status, config JSONB
  • metrics, last_run_at, last_error

agent_runs

Ажиллагааны лог.

  • trigger_kind, status, summary
  • tokens_used, metrics, error
  • owner read-only RLS

shop_dashboard_layouts

Widget самбарын байрлал.

  • layout JSONB (widget array)
  • UNIQUE(shop_id, user_id)
  • байхгүй бол default layout

RLS: owner зөвхөн өөрийн shop-ийн мөрийг. Dispatcher service role-оор ажиллаж scoping-ийг handler өөрөө хийнэ. Backfill: идэвхтэй shop бүрд chatbot (+whitelist-аар comment) мөр → одоогийн зан төлөв хадгална.

Эрсдэл ба хяналт

Юуг хамгаалах вэ

💸 Token / зардал

  • Scheduled LLM/image ажил token pool чимээгүй шавхах
  • → Finance/Inventory deterministic (0 кредит)
  • → Generative бүр persistTokenUsage tag; шавхвал run = skipped
  • → Frequency cap + Marketing хүлээн авагч hard cap

🔒 RLS / аюулгүй байдал

  • Dispatcher service role RLS bypass
  • → Query бүрд shop_id scoping + plan дахин шалга
  • → Cron endpoint CRON_SECRET Bearer
  • → Owner-facing reads RLS-ээр

⏱️ Vercel cron хязгаар

  • Аль хэдийн 7 cron бий — agent тус бүрт нэмэхгүй
  • → Ганц /api/cron/agent-scheduler fan-out
  • → Урт ажил Upstash Redis queue/lock + worker drain

♻️ Backward compat

  • Webhook + process-messages хэвээр (Phase 0–4)
  • Backfill: идэвхтэй shop бүрд chatbot/comment мөр
  • Dashboard layout байхгүй → одоогийн layout fallback
