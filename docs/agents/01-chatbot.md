# 💬 Борлуулагч AI — `chatbot`

> Messenger/IG DM дээр 24/7 ажилладаг AI борлуулагч. Асуултанд хариулж, бараа санал болгож, сагсанд нэмж, захиалга авч, шаардлагатай үед хүн рүү шилжүүлнэ.

**Бүлэг:** 💰 Орлого нэмэгдүүлэгч · **Handler:** `llm` (adapter→`routeToAI`) · **Plan:** `lite` · **Бэлэн байдал:** production-д ажиллаж буй гол feature — каталогт **reframe** хийнэ.

---

## 1. Яагаад (бизнес өвдөлт)
- Үйлчлүүлэгч **шөнө дунд, ажлын бус цагт** мессеж бичдэг — эзэн амжиж хариулдаггүй → үйлчлүүлэгч өрсөлдөгч рүү явдаг.
- **Давтагдсан асуулт** (үнэ хэд вэ? размер байгаа юу? хүргэдэг үү?) өдөрт зуу зуун удаа — эзний цагийг иддэг.
- Хүн ядрах, эвгүй хариулах, удаашрах — AI тогтвортой, эелдэг, хурдан.

## 2. Хэн ашиглах вэ
- **Үндсэн:** үйлчлүүлэгч (мэдэхгүй — AI-тай ярьж байгаагаа).
- **Хяналт:** эзэн/борлуулагч — AI-ийн хариуг хянаж, шаардлагатай үед гар авна (handoff).

## 3. Jobs-to-be-done (үйлчлүүлэгчийн талаас)
- "Энэ бараа **байгаа юу**, **үнэ хэд** вэ?"
- "Надад тохирох **санал болгооч**."
- "**Захиалъя** / сагсанд хийе."
- "**Хүнтэй** ярья (нарийн асуудал)."

## 4. Trigger ба горим
- **event:** ирсэн мессеж (Messenger/IG DM webhook).
- **manual:** каталогийн "Test" — эзэн өөрөө туршина.

## 5. Гол логик урсгал (бэлэн — reframe)
1. Webhook → гарын үсэг шалгах → мессеж хадгалах → shop config ачаалах.
2. **Контекст цуглуулах:** бараа, сагс, үйлчлүүлэгчийн санах ой (`ai_memory`), AgentRole (sales/support г.м.).
3. **Gemini function calling:** бараа хайх, сагсанд нэмэх, захиалга үүсгэх, нөөц шалгах tool-уудаар хариу үүсгэнэ.
4. Хариуг Meta Graph API-аар буцаах.

## 6. Шийдвэрийн цэг ба edge case (өөрөө бодох)
- **Хүн рүү хэзээ шилжүүлэх (handoff):** гомдол, уур, нарийн тохиролцоо, AI эргэлзсэн үед — босго хэрхэн тодорхойлох?
- **Sentiment:** үйлчлүүлэгч ууртай бол өнгө аясыг өөрчлөх, хүн дуудах.
- **Үнийн хэлэлцээ:** "хямдруулж болох уу?" — AI хэр эрх мэдэлтэй (тогтсон бодлого хэрэгтэй)?
- **Бэлэн бус бараа:** "дууссан" гэхээсээ өмнө орлуулах санал, эсвэл "ирэхэд мэдэгдье".
- **Олон хэл/слэнг:** монгол слэнг, латинаар бичсэн монгол — таних.
- **Буруу мэдээлэл (hallucination):** байхгүй барааг "байна" гэхгүй байх — tool үр дүнд л тулгуурлах.

## 7. Codebase reuse
- `src/lib/ai/AIRouter.ts` → `routeToAI` — гол урсгал.
- `src/lib/ai/ToolExecutor.ts`, `src/lib/ai/tools/` — function calling.
- `src/lib/ai/agents/` → `AgentRole`/`AgentRoleDefinition` (дотоод давхарга).
- AI Settings (`/dashboard/ai-settings`) — өнгө аяс, дүрэм.
- `src/app/api/webhook/route.ts` — webhook урсгал.

## 8. Шинээр барих (reframe тул бага)
- Каталогийн card + run history + "Test".
- Adapter: chatbot handler → `routeToAI` нимгэн дуудлага (dispatcher-тэй холбох).
- Handoff/sentiment-ийн илүү тодорхой логик (хүсвэл).

## 9. Config (shop тус бүр)
- AgentRole (борлуулалт/мэдээлэл/дэмжлэг/lead).
- Өнгө аяс, мэндлэх үг, ажлын цаг.
- Handoff босго, хориглосон сэдэв.
- Үнийн хэлэлцээний эрх.

## 10. Widget
- `aiProgress` — AI-ийн өнөөдрийн хариу/хөрвүүлэлт.
- `tokenBreakdown` — token хэрэглээ.

## 11. Кредит / зардал
- **1 кредит / LLM хариу** (мессеж). Энэ нь хэрэглээний кредит загварын гол хэрэглэгч.

## 12. Амжилтын хэмжүүр (KPI)
- Хариу өгөх хурд, хариулсан мессежийн хувь.
- Чатаас захиалга болсон хувь (conversion).
- Handoff хувь (хэт өндөр бол AI муу, хэт бага бол эрсдэлтэй).
- Үйлчлүүлэгчийн сэтгэл ханамж.

## 13. Сайжруулах санаанууд (v2+)
- Бусад agent-тай гүн холбоо: чатаас шууд төлбөр (payment), VIP таних (CRM).
- Дуу хоолой/voice мессеж таних.
- Проактив follow-up ("өчигдөр асуусан бараа тань ирлээ").
- Олон хэл (англи жуулчин).

## 14. Эрсдэл / Монгол-онцлог анхаарал
- **Hallucination = алдагдсан итгэл/буруу захиалга** → зөвхөн tool үр дүнд тулгуурла.
- Монгол хэл, слэнг, латин бичлэг таних чанар.
- Webhook халуун зам — **reframe хийхдээ энэ урсгалыг хөндөхгүй** (backward compat чухал).
