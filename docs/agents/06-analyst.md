# 📊 Бизнес аналист — `analyst`

> Дата шинжээч биш эзэнд "юу болж байна, яагаад, дараа нь юу хийх вэ"-г ойлгомжтой монгол хэлээр өдөр бүр хэлж өгдөг agent. Тоо биш — **үйлдэл санал болгоно**.

**Бүлэг:** 🏆 Ажил хөнгөлөгч (back-office) · **Handler:** `deterministic` (0 token, заавал биш LLM өнгөлгөө) · **Plan:** `pro`

---

## 1. Яагаад (бизнес өвдөлт)
- Дашбоард дээр тоо бий ч **"тэгээд яахав?"** гэдэг нь тодорхойгүй. Эзэн график хараад дүгнэлт гаргаж чаддаггүй.
- Борлуулалт **яагаад буурсныг** мэддэггүй (улирал? бараа дууссан? сурталчилгаа зогссон?).
- Аль бараа ашигтай, аль цаг/гараг идэвхтэй, аль үйлчлүүлэгч үнэ цэнтэйг **анзаардаггүй**.
- Эзэн өдөр бүр өөрөө анализ хийх цаг, чадвар байхгүй.

Энэ agent нь "хувийн бизнес зөвлөх" шиг — өдөр бүр товч, ойлгомжтой, **үйлдэлд хөтөлсөн** дүгнэлт өгнө.

## 2. Хэн ашиглах вэ
- **Үндсэн:** бизнес эзэн (өдөр бүр 1 минутад дүр зургаа харах).
- **Хоёрдогч:** менежер — багийн KPI хянах.

## 3. Jobs-to-be-done
- "Өнөөдөр **юу болсныг** товч хэлж өг."
- "Борлуулалт яагаад **буурав**?"
- "**Юунд анхаарах** ёстой вэ — дараагийн алхам?"
- "Аль бараа/цаг/үйлчлүүлэгч хамгийн **ашигтай** вэ?"

## 4. Trigger ба горим
- **schedule:** өдөр бүр өглөө (өчигдрийн нэгтгэл + insight), 7 хоног/сар бүр (томоохон тойм).
- **manual:** эзэн "Одоогийн байдлаар тайлан" дарна.
- **event (заавал биш):** онцгой үйл явдал (рекорд борлуулалт, огцом уналт) → шуурхай мэдэгдэл.

## 5. Гол логик урсгал (анхны таамаг)
1. **Нэгтгэх:** өдрийн борлуулалт, захиалгын тоо, дундаж захиалга (AOV), шинэ vs давтан үйлчлүүлэгч, хөрвүүлэлтийн юүлүүр (`conversion_funnel`).
2. **Чиг хандлага:** өмнөх өдөр/долоо хоног/сартай харьцуулах (өсөлт/уналт %).
3. **Аномали илрүүлэх:** хэвийн хэлбэлзлээс гажсан зүйл ("Лхагва гарагт 30% буурсан", "X бараа гэнэт огцом зарагдсан").
4. **Insight үүсгэх:** `SmartInsights` логикийг өргөтгөж — top sellers, бараа төрлийн төвлөрөл, AOV сэрэмж г.м.
5. **Үйлдэл санал:** insight бүрд "юу хийх" зөвлөмж ("X бараа дуусч байна → дахин захиал", "Y цагт идэвхтэй → тэр үед пост тавь").
6. **Хүргэх:** натурал монгол хэлээр товч (заавал биш Gemini-ээр өнгөлсөн) push + widget.

## 6. Шийдвэрийн цэг ба edge case (өөрөө бодох)
- **Аль хэмжүүр чухал вэ?** Бизнес төрлөөр өөр (хувцас vs хоол vs үйлчилгээ). Тэргүүлэх KPI-г хэрхэн сонгох?
- **Аномали босго:** хэдэн % хазайхад "анхаарах" гэж тооцох (хуурамч сэрэмжээс зайлсхийх)?
- **Жижиг дата:** өдөрт 2-3 захиалгатай shop-д статистик утгагүй → хэзээ "дата хүрэлцэхгүй" гэж хэлэх вэ?
- **Зөвлөмжийн тэргүүлэл:** олон insight гарвал хамгийн чухал 3-ыг л харуулах (мэдээллийн ачаалал бууруулах).
- **Шалтгаан тайлбар:** уналтын шалтгааныг таамаглах нь эрсдэлтэй — баттай дата байхгүй бол "магадгүй" гэж зөөлрүүлэх.

## 7. Codebase reuse
- `src/components/dashboard/SmartInsights.tsx` — **бэлэн** монгол insight (орлогын чиг, top sellers, бараа төрөл, AOV сэрэмж).
- `supabase/migrations/20260120100000_ai_analytics_tables.sql` — `ai_analytics`, `conversion_funnel`, `get_ai_metrics_summary()`.
- `src/app/api/dashboard/stats/route.ts` — өдөр/долоо хоног/сар bucket нэгтгэл.
- `src/app/api/dashboard/ai-stats/route.ts` — AI хэрэглээний breakdown.

## 8. Шинээр барих
- Бизнес-түвшний KPI: LTV (customer lifetime value), cohort, retention.
- Аномали илрүүлэх engine (баримжаа статистик).
- Үйлдэлд хөтөлсөн зөвлөмж генератор (insight → action).
- Өдрийн авто push тайлан.

## 9. Config (shop тус бүр)
- Өдрийн тайлан авах эсэх + цаг.
- Тэргүүлэх KPI сонголт (борлуулалт / ашиг / шинэ үйлчлүүлэгч).
- Аномали мэдэгдэл авах эсэх + мэдрэг байдал.

## 10. Widget
- `agent.analyst.dailyReport` — өчигдрийн нэгтгэл + чиг сум + гол insight.
- `agent.analyst.insights` — тэргүүлэх 3 зөвлөмж картууд.

## 11. Кредит / зардал
- **0 кредит** — deterministic тооцоо + бэлэн `SmartInsights`. Натурал хэлээр өнгөлөх (LLM) сонголттой бол бага кредит — гэхдээ template-ээр ихэнхийг шийднэ.

## 12. Амжилтын хэмжүүр (KPI)
- Өдрийн тайлан нээх/уншсан хувь (daily active).
- Зөвлөмжөөс үйлдэл хийсэн хувь (зөвлөмж дарж → бараа захиалсан г.м.).
- Эзний "бизнесээ илүү ойлгодог болсон" feedback.

## 13. Сайжруулах санаанууд (v2+)
- Бусад agent-тай холбох: insight → шууд үйлдэл (Boost-д "энэ барааг сурталч", CRM-д "эдгээр VIP-д хандах").
- Таамаглал: дараа сарын борлуулалт, мөнгөн урсгал.
- Бенчмарк: ижил салбарын дундажтай харьцуулах (анонимаар).
- "Асуу" горим: эзэн чөлөөт асуулт асууж дата дээр хариу авах (LLM + SQL).
- Зорилго тавих + явц хянах ("энэ сар 5 саяын борлуулалт", явцыг хэлж байх).

## 14. Эрсдэл / Монгол-онцлог анхаарал
- **Буруу дүгнэлт итгэл алдагдуулна:** баримтгүй шалтгаан таамаглавал эзэн итгэхээ болино → магадлалт хэллэг.
- Жижиг дата дээр статистик хий хоосон → "дата хүрэлцэхгүй" гэж шударгаар хэлэх.
- Мэдээллийн ачаалал: хэт олон тоо/зөвлөмж → эзэн уншихаа болино. Энгийн, цөөн, тодорхой байх.
