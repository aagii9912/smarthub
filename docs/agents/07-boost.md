# 🚀 Boost оновчлогч — `boost`

> "Boost" товч дарж мөнгө шатаахаа болиод, аль бараа сурталчлахаа AI-аар тооцож, төсөв/ROAS таамаглаж, зар бичвэр+зураг хүртэл гаргаж өгдөг agent.

**Бүлэг:** 🏆 Ажил хөнгөлөгч (back-office) · **Handler:** `hybrid` (Meta Ads + креатив) · **Plan:** `enterprise`
**⚠️ Бэлэн байдал: 0% — БҮРЭН ШИНЭ.** Meta Marketing API интеграц одоо огт алга (зөвхөн organic messaging). Хамгийн том gap → roadmap-ийн **хамгийн сүүлд** (Phase 5).

---

## 1. Яагаад (бизнес өвдөлт)
- Эзэн "Boost post" товч дараад **санамсаргүй** мөнгө зарцуулдаг, үр дүнг (ROAS) **хэмждэггүй** — ашигтай юу, дэмий юу мэдэхгүй.
- **Аль бараа/постыг** сурталчлахаа мэдэхгүй — аль хэдийн органикаар сайн хөрвүүлж буй "winner"-ийг хайдаггүй.
- Зар бичвэр, креатив зураг бэлдэхэд **цаг, ур чадвар** дутдаг → муухай зар → үр дүн муу.
- Audience (хэнд харуулах) тохиргоог ойлгодоггүй.

Энэ agent нь "хямд диджитал маркетер" шиг — өгөгдөл дээр тулгуурлан **юуг, хэнд, хэдэн төгрөгөөр** сурталчлахыг хэлж, креативыг нь хүртэл бэлдэнэ.

## 2. Хэн ашиглах вэ
- **Үндсэн:** эзэн / маркетинг хариуцсан ажилтан.
- Ихэвчлэн томоохон (enterprise) бизнес — сурталчилгааны бодит төсөвтэй.

## 3. Jobs-to-be-done
- "Аль барааг сурталчилбал **ашигтай** вэ?"
- "Хэдэн төгрөгийн **төсөв** тавих вэ, ямар **үр дүн** хүлээх вэ?"
- "Зар бичвэр, зургийг **надад зориулж гаргаж өг**."
- "Сүүлийн boost-ууд **ашигтай байсан уу** (ROAS)?"

## 4. Trigger ба горим
- **schedule:** долоо хоног бүр органик winner шинжилж boost зөвлөмж бэлдэх; идэвхтэй кампанит ажлын ROAS хянах.
- **manual:** эзэн "Сурталчлах бараа санал болго", "Зар гаргаж өг" дарна.

## 5. Гол логик урсгал (анхны таамаг)
1. **Winner илрүүлэх:** органик гүйцэтгэлээс (`conversion_funnel`, view→cart→order) хамгийн сайн хөрвүүлж буй бараа/постыг олно.
2. **Boost зөвлөмж:** "Энэ барааг сурталч → санал төсөв X төгрөг → таамаг хүрэх хүн / захиалга / ROAS".
3. **Audience санал:** одоо байгаа үйлчлүүлэгчийн профайл (`customers.ai_memory`, бүс) дээр суурилж target.
4. **Креатив үүсгэх:** зар бичвэр (LLM) + бүтээгдэхүүний зар зураг (`GeminiImageProvider`) — брэндийн өнгө/логотой.
5. **(v2) Шууд хүргэх:** Meta Marketing API-аар кампанит ажил үүсгэх/удирдах.
6. **ROAS хянах:** Meta-аас зарцуулалт/үр дүн татаж, орлоготой тулгаж ROAS бодох → "энэ кампанит ажил ашигтай/алдагдалтай".

## 6. Шийдвэрийн цэг ба edge case (өөрөө бодох)
- **Boost-д тэнцэх босго:** ямар бараа "winner" — хамгийн их зарагдсан уу, хамгийн өндөр маржинтай юу, эсвэл хослол уу?
- **Төсвийн дээд хязгаар:** AI хэт их төсөв санал болговол эзэн мөнгөө шатаана → hard cap + эзэн баталгаажуулна.
- **ROAS таамаг найдвартай юу?** Органик гүйцэтгэл → төлбөртэйд шилжих нь шууд биш → "таамаг" гэж тэмдэглэ.
- **Meta API эрх/токен:** ad account холболт, эрхийн хүрээ, billing — нарийн OAuth урсгал.
- **Креативын чанар/брэнд:** AI зураг буруу/чанаргүй гаргаж болзошгүй → эзэн заавал хянаж баталгаажуулна.
- **Бодлогын зөрчил:** Meta зар бодлого (хориотой бараа, текстийн хэмжээ) — татгалзвал яах.

## 7. Codebase reuse
- `src/lib/ai/providers/AIProvider.ts` → шинэ **сонголтот** `ImageGenProviderInterface` (зураг үүсгэх).
- `supabase/migrations/20260120100000_ai_analytics_tables.sql` → `conversion_funnel` (winner илрүүлэхэд).
- `src/lib/facebook/messenger.ts` → Graph API хандах загвар (Marketing API өргөтгөхөд лавлах).
- `persistTokenUsage` → креатив зардлыг `'boost_creative'` tag-аар бүртгэх.

## 8. Шинээр барих (ихэнх нь шинэ)
- **Meta Marketing API интеграц** (ad account OAuth, кампанит ажил CRUD, insights/ROAS татах) — **бүрэн шинэ, том ажил**.
- `src/lib/ai/providers/GeminiImageProvider.ts` (Gemini image model).
- Winner илрүүлэх + ROAS таамаглах logic.
- Зар бичвэр + креатив үүсгэгч.
- (магадгүй) ad-метрик cache хүснэгт.

## 9. Config (shop тус бүр)
- Meta ad account холболт + billing.
- Сарын/кампанит ажлын төсвийн дээд хязгаар.
- Брэндийн өнгө/лого/өнгө аяс (креативт).
- Target бүс/насны хүрээ.

## 10. Widget
- `agent.boost.roas` — идэвхтэй кампанит ажил, зарцуулалт vs орлого, ROAS.
- `agent.boost.creatives` — үүсгэсэн зар зураг/бичвэрийн галерей.

## 11. Кредит / зардал
- Boost оновчлол **зөвлөмж** = **5 кредит / шинжилгээ** (winner анализ + таамаг).
- Креатив зураг = `boost_creative` tag, зураг тутамд кредит (хамгийн өндөр зардал).
- ⚠️ Meta-д төлөх **зар төсөв нь Syncly кредитээс тусдаа** — эзэн өөрийн Meta billing-ээр төлнө. Үүнийг тодорхой ялгах.

## 12. Амжилтын хэмжүүр (KPI)
- Дундаж ROAS (сурталчилгааны 1₮ → хэдэн ₮ орлого).
- Зар бэлдэх хугацаа (AI-аар хэдэн дахин хурдан).
- Boost зөвлөмжөөс хийсэн кампанит ажлын хувь ба амжилт.

## 13. Сайжруулах санаанууд (v2+)
- A/B тест авто (`ab_experiments` хүснэгт бэлэн) — олон креатив тулгах.
- Retargeting: сагс орхисон/үзсэн хүн рүү авто зар (CRM-тэй холбоо).
- Олон суваг (FB/IG/мессенжер) бюджет авто хуваарилалт.
- Улирал/баярын автомат кампанит ажил.
- Lookalike audience (одоогийн худалдан авагч дээр суурилсан).

## 14. Эрсдэл / Монгол-онцлог анхаарал
- **Том техникийн ажил:** Meta Marketing API нь OAuth, billing, бодлогын review — энэ agent-ийг сүүлд (Phase 5) хийх гол шалтгаан.
- **Мөнгөн эрсдэл:** буруу тохиргоо эзний мөнгийг шатаана → төсвийн hard cap + заавал баталгаажуулалт.
- Монгол зах зээлд Meta зарын CPM хямд ч, target нарийвчлал хязгаарлагдмал → таамаг болгоомжтой.
- Зар төсөв vs кредит хоёрыг **тодорхой ялгахгүй** бол эзэн будлиулна.
