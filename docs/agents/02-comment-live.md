# 📣 Comment & Live оператор — `comment`

> FB/IG пост, тусгайлан **Live худалдаа**-ны доорх сэтгэгдэлд авто хариулж, "+1"/размер/өнгө зэрэг захиалгын дохиог барьж DM руу хөрвүүлдэг agent.

**Бүлэг:** 💰 Орлого нэмэгдүүлэгч · **Handler:** `deterministic`/`hybrid` · **Plan:** `starter` · **Бэлэн байдал:** comment automation бий — Live capture-аар **reframe+enhance**.

---

## 1. Яагаад (бизнес өвдөлт)
- **Live худалдаа** Монголд асар түгээмэл. Live үед зуу зуун коммент ("+1", "M размер", "хар өнгө", "үнэ?") — эзэн **гараар бариад огт амждаггүй**, захиалга алдагдана.
- Пост бүрийн доор "үнэ хэд?", "inbox-доо хар" гэсэн давтагдсан коммент — гараар хариулах ажилгүй цаг.
- Хариу удаан → үйлчлүүлэгч хүйтэрнэ. Бусдын өмнө үнэ асуусан комментод хариулахгүй бол борлуулалт алдана.

## 2. Хэн ашиглах вэ
- **Үндсэн:** үйлчлүүлэгч (коммент бичигч).
- **Хяналт:** эзэн/борлуулагч — Live үеэр захиалгын урсгалыг хянана.

## 3. Jobs-to-be-done
- Үйлчлүүлэгч: "Коммент бичээд **хурдан хариу** авах", "**+1 гээд захиалга өгөх**".
- Эзэн: "Live дээрх **бүх захиалгыг алдалгүй барих**", "Давтагдсан асуултад **автоматаар** хариулах".

## 4. Trigger ба горим
- **event:** шинэ коммент (FB/IG webhook).
- Live горим: тухайн Live/пост идэвхтэй үед эрчимтэй capture.

## 5. Гол логик урсгал (анхны таамаг)
1. Коммент webhook → текстийг intent/keyword-оор ангилах (үнэ асуусан / захиалга / асуулт / spam).
2. **Нийтэд хариу** (template, "inbox-доо хараарай") эсвэл **DM trigger** (хувийн хариу/каталог).
3. **Live "+1" capture:** "+1", тоо, размер (S/M/L/XL), өнгө зэргийг таньж → тухайн барааны захиалга үүсгэх дохио → DM рүү хөтөлж төлбөр/хаяг авах.
4. Захиалгыг хүн рүү (борлуулагч) эсвэл chatbot руу дамжуулж баталгаажуулна.

## 6. Шийдвэрийн цэг ба edge case (өөрөө бодох)
- **Аль постод идэвхтэй:** бүх постод уу, зөвхөн идэвхжүүлсэн/Live-д уу?
- **Нийтэд vs хувийн:** үнийг нийтэд хэлэх үү (өрсөлдөгч хардаг), DM-д л хэлэх үү?
- **Live "+1" тайлал:** "+1" аль барааг хэлж байна (Live-ийн тухайн агшны бараатай холбох — цаг хугацааны контекст)? Энэ хамгийн төвөгтэй.
- **Размер/өнгө parse:** "М-ийг хараар 2ш" гэх мэт нийлмэл захиалга.
- **Spam/муухай үг** шүүх, давхар коммент нэг хүнээс.
- **Хэл:** латинаар бичсэн монгол ("une hed ve").

## 7. Codebase reuse
- `CommentAutomationService` (comment automation бэлэн логик).
- `src/app/api/webhook/route.ts` доторх comment салбар.
- `src/lib/facebook/messenger.ts` — нийтэд хариу / DM илгээх.
- `supabase/migrations/20260301_comment_automations.sql` — `comment_automations`.
- `COMMENT_AUTOMATION_ENABLED_SHOPS` whitelist → `shop_agents` руу migrate.

## 8. Шинээр барих
- **Live "+1"/тоо/размер/өнгө capture** логик (гол шинэ үнэ цэн).
- Live агшин ↔ бараа холбох контекст.
- whitelist-ийг `shop_agents`-д шилжүүлэх.

## 9. Config (shop тус бүр)
- Идэвхтэй постын хамрах хүрээ.
- Keyword → хариу template-ууд.
- Үнийг нийтэд хэлэх эсэх.
- Live горим тохиргоо (идэвхжүүлэх бараа).

## 10. Widget
- `automation` метрик — хариулсан коммент, DM хөрвүүлэлт, Live захиалга.

## 11. Кредит / зардал
- **1 кредит / LLM хариу** (intent/хувийн хариу LLM шаардвал). Template-д суурилсан энгийн хариу = 0.

## 12. Амжилтын хэмжүүр (KPI)
- Хариулсан комментын хувь, дундаж хурд.
- Live дээр баригдсан "+1" захиалгын тоо (алдагдсан биш).
- Коммент → DM → захиалга хөрвүүлэлт.

## 13. Сайжруулах санаанууд (v2+)
- Live дээр бодит цагт нөөц шинэчлэх (inventory-тэй холбоо) — "дууссан" гэж шуурхай мэдэгдэх.
- Live-ийн дараа авто нэгтгэл: хэн юу захиалсан жагсаалт → багц боловсруулалт.
- Видео/зураг коммент таних.
- Хүлээлгийн жагсаалт (дууссан бараанд "+1" хийсэн хүн рүү ирэхэд мэдэгдэх).

## 14. Эрсдэл / Монгол-онцлог анхаарал
- **Live "+1" контекст** — Монголын Live худалдааны хамгийн өвөрмөц, төвөгтэй хэсэг. Тухайн агшинд аль бараа гарч байсныг зөв холбохгүй бол буруу захиалга.
- Олон зуун коммент богино хугацаанд → rate/ачаалал.
- Латин монгол, слэнг, эмодзи taclil.
- Худал захиалга (зүгээр "+1" гээд аваагүй) — баталгаажуулалт хэрэгтэй.
